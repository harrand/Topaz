Detailed Description

High-level object used to render geometry.

Renderers can be described as follows:

  • A Renderer has zero or more IRendererInput. It is planned to allow Renderers without inputs (for example, if vertices are hard-coded within the shader) but this is not yet implemented.
    • Drawing multiple inputs is batched as much as possible. Typically, a renderer will emit 1-2 draw calls, regardless of the number of inputs.
  • A Renderer has exactly one IRendererOutput.
  • Renderers will never change their input or its corresponding input data.
  • Renderers cannot have their input/output changed. However, dynamic input data is supported via IRendererDynamicInput.
  • Renderers have an initial clear-colour but this can be changed after construction. If it is changed during rendering, this may reconstruct the entire render pipeline and incur a large spike in latency.
  • Renderers will be able to use any number of resources (aka uniform buffers, textures etc...) This is not yet implemented.
  • Renderers will be able to edit data for a given resource if the resource is writable (e.g SSBOs).

◆ get_input()

virtual IRendererInput* tz::gl::IRenderer::get_input ( RendererInputHandle  handle)
pure virtual

Retrieve the renderer input.

Note
Each renderer takes a copy of the input it was given in its corresponding IRendererBuilder. This is NOT the same input as the one you gave to the builder.
The pointer returned is valid until this Renderer reaches the end of its lifetime.

For static renderer inputs, you will rarely find this useful. If you have dynamic renderer inputs, you should retrieve a pointer to the input here and edit its data as you wish.

Returns
IRendererInput* pointing to the renderer's input.

virtual void tz::gl::IRenderer::render ( )
pure virtual

Proceed through the provided render-pass using any inputs and resources.

Note
This function does not take in a draw list. This means that it will use the most-recently-provided draw list. If no draw list was ever provided, then a draw list equivalent of once-per-input will be used.
