Detailed Description

A collection of low-level renderer-agnostic graphical interfaces.

Typedef Documentation

◆ RendererDrawList

using tz::gl::RendererDrawList = typedef tz::BasicList<tz::gl::RendererInputHandle>

RendererDrawLists contain a permutation of renderer inputs which will be drawn in a given render invocation. Typically, you can use a draw list to represent objects in a 3D scene.

Let's say for example you're building a fantasy game. You would like to draw three goblins within the world. The goblin mesh is stored within a IRenderer as an input, with the input handle x.

You will need to draw the same input x three times. Your draw list may look like: {x, x, x}. Note that this will only draw the goblin mesh three times, and none of the other inputs. The draw list should correspond exactly with every object you would like to draw for the given frame. Do not neglect the concept of draw lists as the default draw-list is rarely useful. See IRenderer::render() for information about default draw lists.

Renderers natively support culling of faces. See IRendererBuilder for defaults.

Enumerator

No faces are culled.
