127 // this is a component of a mesh_renderer. deals with a compute pre-pass to generate draw commands.
128 // this has its own compute pass stored internally. you're gonna want to include this as a member in the mesh renderer and then use its API accordingly.
129 // theres alot of methods with overlapping interests.
130 // generally, when you wanna add N new objects:
131 // just use add_new_draws(N), and
132 // then for i=0,N,++ set_mesh_at(object_id, whichever_mesh_locator_you_want)
133 // please make sure the mesh_locator maps to something the vertex_wrangler gives you... otherwise shit is bound to get corrupted.

142 std::size_t get_draw_capacity() const;
143 // change the draw capacity to something new.
144 // this is always very slow. 2 renderer edits.
145 void set_draw_capacity(std::size_t new_capacity);
146 // set the mesh of an existing draw to something else. does not affect draw count. draw_id needs to be less than draw count.
147 mesh_locator get_mesh_at(std::size_t draw_id) const;

149 bool get_visibility_at(std::size_t draw_id) const;

151 // add N new draws. increments the draw count N times. if draw capacity is too small, increase it by N or double its capacity, whichever is bigger.
152 // mesh at the new draw will be an empty mesh locator. you're free to change it to something else.

202 // objects represent a single renderable thing.
203 // they are comprised of:
204 // a transform (as part of a full transform hierarchy)
205 // - this is a part of the object tree.
206 // - the global transform of the object based on the hierarchy is computed every frame. cpu-side.
207 // a fixed-size array of texture locators. essentially an object can use multiple textures. useful for normal-maps etc...
208 // - this also contains a tint.
209 // - texture locators are part of the object data.
210 // an optional mesh!
211 // - the mesh of object X corresponds to the mesh_locator at index X of the mesh_locator_buffer of the compute_pass.
212 // - an empty (default) mesh_locator means that the object exists in the hierarchy, but isn't meant to be rendered (e.g bones in a skeletal animation)
